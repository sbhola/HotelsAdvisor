Purpose of Style guide

The purpose of this guide is to share the HTML & CSS formatting rules used to create a Mystique Theme. A consistent user experience across the Mystique themes is achieved through a similar interaction and behavior. As an example, when a user is searching for Air, car or a Hotel, the search behaviors and function across different Themes is the same. The look and feel of the result page could be different based on the font face, text and button colors etc.

The Themes will have different layout, placement of widgets on the Home page, but the rest of the flow will be the same.

This guide will help the designers & developers with rules to create a new Theme or modify an existing theme. This Guide also provides standards to ensure that

  • The CSS files don't contain any unused, redundant styles
  • The files/folder structure for all the Themes is the same
  • There are no unused, redundant images and all the icons are grouped as a sprite image, as far as possible.

Audience

Style guide will be referred to whenever new content, new features are introduced, or a new Theme is created. These Guidelines are essential for Web Designers and Web Developers. They must consider these when adding or changing content on Mystique Web Application.

Default Theme Folder

Themes folder has Default Theme folder. It is Mystique's default base theme.

Bootstrap Folder

Default theme folder contain Bootstrap folder with assets and images.

Images Folder

All the Themes will have a standard folder structure for the images.

Separate folder for every module eg. Flight / Hotel / Car / Activity / Emails / Plugnplay. There is also a common folder which contain common images used throughout site.

CSS Folder

There are two main css folder libs and user.

Each of the CSS has a specific role which will be explained in the following sections.

"libs" Folder

This folder contains CSS files for libraries like Bootstrap, jQuery and jQuery plugins.

"user" Folder

This folder contains custom CSS files.

"core-styles.css" is main CSS file.

"z.Custom.css" this file is written by developer team.

core-styles.css

This file contains global CSS classes which impacts the whole site. Only Creative team will make any changes to this file. If a developer has to change anything in "core-style.css" it will have to be reviewed by Creative beforehand.

The top section of "core-style.css" has all the default styles and required for the Theme. E.g. styles for the tags like html, body, custom icons, font size, font color, border.

The class naming convention used is Bootstrap style simple hypenated(-) separated words eg. "class-name"

Common Padding and Margin classes

To avoid repetitive margin and padding specifications in the CSS, common.css has a set of predefined Margins and Padding classes with various values from Small, Medium & Large. By default, the values for these margin/padding are Small=5px, Medium=10px and Large=20px.

A designer/developer should avoid specifying margins and padding while creating new CSS classes and instead append these predefined classes for margin and padding wherever required.

eg. for top padding of 10px, class used is "ptm"

<span class="ptm">top padding of 10px(medium)</span>

eg. for bottom margin of 20px, class used is "mbl"

Common Colors, Font size classes

All the commonly used font sizes and colors are defined.

eg. for font size 16 we need to use class "font-16".

eg. for orange color font we need to use class "color-orange".

font-low-case font-up-case

Custom Icon Sprite

The icon sprite is created online on instantsprite.com with default options.

Sprite Naming Convention

The class name for Flight icon will start with "flight-icon-XXXXX"

The class name for Car icon will start with "car-icon-XXXXX"

The class name for Hotel icon will start with "hotel-icon-XXXXX"

The class name for Activity icon will start with "activity-icon-XXXXX"

If you want to add any new icon you need to include it in existing sprite with the proper naming convention as above.

The icons which are not module specific are distinguished as common

The class name for Common icon will start with "t-icon-XXXXX"

All the icons can be seen module wise listed in Primeface.

z.Custom.css

Mystique developers have created the z custom css file for their convenience.

Mystique team should not create same classes again and again avoid redundant classes.

Default Font Family

Default theme is using Google's Lato font family. No any other font family is used.

The Google font API is linked in the Master Page head section and the font family is mentioned in the "core-style.css".

Google Font API

‹link href='http://fonts.googleapis.com/css?family=Roboto:400,100,100italic,300,300italic,400italic,500,500italic,700,700italic,900,900italic' rel='stylesheet' type='text/css'›

If in case you need to implement any new font family you need the font API as above with different font weights eg. 400, 600, 700 etc.
